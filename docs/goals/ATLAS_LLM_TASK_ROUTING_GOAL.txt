# ATLAS OS GOAL - TASK-AWARE MULTI-LLM ROUTING SYSTEM

## OBJECTIVE

Upgrade Atlas OS from a single active-provider runtime into a task-aware multi-LLM orchestration system.

Atlas must support separate provider and model assignments for:

1. Workhorse processing
2. Research synthesis
3. Final decision synthesis

Primary goals:

- use lower-cost models for repetitive processing
- reserve premium GPT or Claude models for high-value synthesis
- preserve Atlas cognition and decision boundaries
- provide per-task provider, model, fallback, latency, usage, and cost visibility

This Goal is authorized for implementation only after the required Production Trial Issue is recorded and linked.

## CURRENT REPOSITORY TRUTH

The current repository already contains:

- runtime/llm/provider_registry.py
  - provider configuration
  - Keychain-first secret storage
  - health and latency tracking
  - API-backed model discovery

- runtime/llm/provider_router.py
  - explicit provider_id and model
  - provider fallback
  - normalized raw-text response

- runtime/llm_router.py
  - compatibility adapter
  - telemetry integration

- runtime/decision_loop.py
  - validated DecisionPacket path
  - bounded LLM cognitive feedback

- runtime/config/user_config.json
  - local runtime and provider configuration

- ui/pages/settings.py
  - provider management
  - provider health
  - model selection

Current limitation:

Atlas has one global active provider and one global fallback chain. It does not route requests by task role.

### ROLE 1 - WORKHORSE

Responsibilities:

- unstructured text extraction
- classification
- deduplication assistance
- fact and source extraction
- structured Signal / Evidence preparation

Recommended providers:

- GLM
- Kimi
- DeepSeek
- other low-cost OpenAI-compatible providers

Output must be structured, source-aware, and non-authoritative.

It must not produce portfolio actions or modify cognition.

### ROLE 2 - RESEARCH

Responsibilities:

- cross-source comparison
- counter-evidence review
- portfolio relevance analysis
- causal-chain drafting
- hypothesis comparison
- uncertainty identification

Output is a research input to the final decision stage.

It must not bypass Seven Layer Reasoning, CDE, or Decision Contract.

### ROLE 3 - DECISION

Responsibilities:

- final Decision Brief synthesis
- final risk and scenario presentation
- user-facing Ask Atlas response
- validated DecisionPacket generation

Recommended providers:

- MoreCode / GPT
- OpenAI
- Claude

All output must pass through the existing Decision Contract and validator.

Only this role may feed the existing bounded LLM cognitive feedback path.

## RUNTIME CADENCE

Preserve the current 60-second daemon tick.

A tick must not automatically equal an LLM request.

Required policy:

Every 60 seconds:
    check runtime state, inbox, schedules, and source freshness

When new unstructured evidence exists:
    optionally run Workhorse

Every configured proactive cycle, default 2 hours:
    run Research only when relevant evidence or context exists

When a material brief is required or the user asks Atlas:
    run Decision synthesis

When there is no meaningful input delta:
    do not call an LLM

Use existing event hashes, timestamps, source freshness, and runtime state where possible. Do not introduce a new prediction algorithm to determine materiality.
