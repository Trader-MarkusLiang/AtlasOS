# Case: HBM Supercycle

Expected Atlas output:

- Strengthen Memory as a first-level bottleneck.
- Reinforce Equipment and Materials because HBM capacity expansion increases process intensity.
- Track whether HBM pricing, capacity tightness, and advanced packaging constraints transmit into earnings.
- Do not treat the word "supercycle" as a direct trading action.
- If the thesis is right but price is extended, wait for reset instead of chasing.

## Seven-Layer Reasoning

| Layer | Reading |
|---|---|
| Fact | HBM demand rises with AI accelerator demand. |
| Physics | Memory bandwidth and capacity become key AI system constraints. |
| Engineering | HBM expansion requires advanced DRAM, packaging, testing, and yield control. |
| Economics | Tight supply can improve pricing and returns for qualified suppliers. |
| Finance | Earnings upside depends on price, volume, capex burden, and yield. |
| Capital | Capital favors Memory first, then Equipment and Materials exposed to HBM expansion. |
| Trading | Core Memory exposure is supported, but entry still depends on risk/reward. |
