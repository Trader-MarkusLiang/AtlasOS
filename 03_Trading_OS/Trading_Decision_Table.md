# Trading Decision Table

Every trading action must include:

1. Action
2. Confidence
3. Logic Chain
4. Evidence
5. Risk / Reward
6. Trigger
7. Counter Argument
8. Review Plan

## Template

| Field | Content |
|---|---|
| Action | Hold / Watch / Add / Reduce / Exit |
| Confidence | Low / Medium / High, based on evidence quality and reasoning-chain strength |
| Logic Chain | Fact -> Physics -> Engineering -> Economics -> Finance -> Capital -> Trading |
| Evidence | Link to case, source note, current-state file, order evidence, or price transmission record |
| Risk / Reward | Expected upside driver, downside risk, and position-size implication |
| Trigger | Price reset, order confirmation, ROI improvement, bottleneck rank change, or invalidation event |
| Counter Argument | Strongest reason the action could be wrong |
| Review Plan | Date or event that forces review |

## Minimum Acceptance Rule

A trading action is incomplete unless all eight fields are filled. If any field is unknown, the
default action is Watch.
